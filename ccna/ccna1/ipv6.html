1. Lectura de direcciones IPv6

Este bloque concentra lo esencial para leer, expandir y abreviar direcciones sin repartir la misma teoría en tres secciones distintas.

Resumen fácil IPv6 usa 128 bits, 8 hextetos y 32 símbolos hexadecimales. En una LAN típica, /64 separa 64 bits de red y 64 bits de interfaz.
128 bits La dirección completa siempre tiene 8 hextetos y cada hexteto representa 16 bits.
Compresión Quitamos ceros a la izquierda y solo usamos :: una vez por dirección.
/64 Es el prefijo clave para SLAAC porque deja otros 64 bits para el ID de interfaz.
AAAA El DNS publica direcciones IPv6 globales en registros AAAA ligados a GUA.

Reglas que no fallan

  • 1 hexteto = 16 bits y 8 hextetos = 128 bits.
  • Los grupos se separan con : y el prefijo se expresa con /n.
  • Primero quitamos ceros a la izquierda; después comprimimos si conviene.
  • La compresión con :: solo puede aparecer una vez y luego se conecta con /64 y SLAAC.

Errores que eliminamos

  • Usar :: dos veces en la misma dirección.
  • Expandir una dirección y quedarse con menos de 8 hextetos.
  • Confundir prefijo de red con ID de interfaz.
  • Memorizar ejemplos sin entender el patrón de los grupos.

Ejemplo resuelto 1: expansión correcta

Dirección: FE80::1

Paso 1: contamos los grupos visibles y detectamos la secuencia omitida.

Paso 2: reponemos 0000 hasta volver a 8 hextetos.

Resultado: FE80:0000:0000:0000:0000:0000:0000:0001

Ejemplo resuelto 2: abreviación correcta

Dirección: 2001:0DB8:000A:0000:0000:0000:0000:0001

Paso 1: quitamos ceros a la izquierda en cada hexteto.

Paso 2: comprimimos la corrida más larga de 0000.

Resultado: 2001:DB8:A::1

Asignación global y DNS AAAA

  • Las GUA se mueven dentro de 2000::/3.
  • La IANA delega bloques a los RIR; luego pasan a operadores y clientes.
  • Cuando buscamos IPv6 público en DNS, el registro que nos interesa es AAAA.

2. Clasificación de direcciones IPv6

Aquí concentramos la taxonomía del material para distinguir unicast, multicast y anycast sin repetir tablas y diagramas.

Resumen fácil GUA = pública y enrutable. LLA = mismo enlace. ULA = uso local. Multicast = grupos. Anycast = misma dirección, responde el nodo más cercano.

Mapa rápido de clasificación

Toca un nodo del mapa para filtrar directamente las tarjetas de Unicast, Multicast o Anycast.

Mostrando todas las tarjetas.

Unicast global (GUA)

2000::/3

Dirección enrutable en Internet IPv6. Es la referencia cuando hablamos de dirección global.

Global y enrutable

Enlace local (LLA)

FE80::/10

Existe por enlace, no se enruta globalmente y participa en vecindad y descubrimiento.

Mismo enlace

Local única (ULA)

FC00::/7

Sirve para direccionamiento interno sin depender de una GUA.

Uso local

Direcciones especiales

::1/128 y ::

::1 prueba interna del host. :: indica ausencia de dirección configurada.

Prueba y ausencia

IPv4 integrada

Usa los 32 bits inferiores para transportar una IPv4 dentro de una dirección IPv6.

Transicion

Multicast general

FF00::/12

Nunca se usa como origen y reemplaza la idea de broadcast general de IPv4.

Uno a muchos

Grupos que sí memorizamos

FF02::1, FF02::2, FF02::5, FF02::A, FF02::1:FF00::/104

Todos los nodos, todos los routers, OSPFv3, EIGRP para IPv6 y nodo solicitado.

Grupos clave

Anycast

Usa rango unicast y la red entrega el tráfico al nodo más cercano.

Nodo mas cercano

Cómo no confundir multicast y anycast

  • Multicast: una dirección llega a muchos receptores del grupo.
  • Anycast: varios nodos anuncian la misma dirección, pero responde el más cercano.
  • Nodo solicitado: es multicast y sale de una unicast; luego aparece otra vez en vecindad.

Patrón rápido de lectura

  • 2000::/3 te orienta a GUA.
  • FE80::/10 te orienta a enlace local.
  • FF00::/12 te orienta a multicast.
  • Anycast no tiene prefijo exclusivo; se entiende por el escenario.

3. Autoconfiguración y vecindad

En este bloque dejamos solo el flujo que realmente necesitas entender: RS, RA, SLAAC y su relación con la vecindad IPv6.

Resumen fácil El host envía RS, el router responde con RA, el prefijo suele ser /64 y el equipo completa su ID de interfaz para formar la GUA.

Flujo mínimo

  • RS lo envía el host para descubrir routers IPv6.
  • RA lo envía el router para anunciar la red.
  • Con ese RA, el host forma su GUA y aprende cómo salir de la LAN.

Qué puede traer un RA

  • Prefijo de red y longitud de prefijo.
  • Gateway predeterminado.
  • DNS y nombre de dominio.

Relación con vecindad

  • La LLA sigue siendo clave porque todo empieza en el mismo enlace.
  • La multicast de nodo solicitado participa en resolución de vecinos.
  • IPv6 evita el broadcast clásico y usa grupos más específicos.

Secuencia mental que conviene repetir

  1. Identificamos el prefijo anunciado.
  2. Comprobamos que el escenario usa /64.
  3. Relacionamos RS con descubrimiento y RA con información de red.
  4. Recordamos que la vecindad usa nodo solicitado en lugar de broadcast.

4. Cisco IOS mínimo

La idea operativa se resume a una secuencia corta y a una verificación crítica: si falta ipv6 unicast-routing, el diseño no se comporta como esperas.

Resumen fácil Si el host no sale de la LAN, primero revisamos ipv6 unicast-routing, la dirección de la interfaz, la LLA y el estado de la interfaz.

Checklist mínima

Cisco IOS 
R1> enable
R1# configure terminal
R1(config)# ipv6 unicast-routing
R1(config)# interface g0/0
R1(config-if)# ipv6 address 2001:DB8:1::1/64
R1(config-if)# no shutdown

Qué revisar cuando algo falla

  • ¿Está activo ipv6 unicast-routing?
  • ¿La interfaz tiene dirección IPv6 correcta?
  • ¿La interfaz está levantada?
  • ¿El host tiene LLA y aprendió el prefijo correcto?

5. Resumen para práctica y examen

Este cierre fusiona glosario y memorización para dejar un repaso corto, útil y sin repeticiones innecesarias.

GUA

Unicast global. Rango: 2000::/3. Ver clasificación.

LLA

Dirección de enlace local. Rango: FE80::/10. Ver vecindad.

ULA

Dirección local única para uso interno. Rango: FC00::/7.

SLAAC

Autoconfiguración sin estado apoyada en RA y prefijos /64.

RS y RA

RS lo envía el host; RA lo responde el router con la información de red. Ver flujo.

Nodo solicitado y anycast

La multicast de nodo solicitado resuelve vecindad; anycast entrega al nodo más cercano. Ver tipos.

Formato: 128 bits, 8 hextetos y 32 símbolos hexadecimales.
Compresión: ceros a la izquierda fuera; :: una sola vez.
Prefijo clave: /64 para SLAAC y separación red/interfaz.
Rangos que sí conviene recordar: 2000::/3, FE80::/10, FC00::/7, FF00::/12.
Operación: host = RS, router = RA, Cisco IOS = ipv6 unicast-routing.

Si podemos explicar en voz alta una dirección completa, distinguir GUA y LLA, y narrar RS → RA → SLAAC, ya estamos listos para el repaso rápido y la práctica.
