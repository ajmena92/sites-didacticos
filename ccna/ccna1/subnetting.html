CCNA 1 / Subnetting y ANDing

🧮 Subnetting y ANDing

Domina el método de la wildcard: AND para encontrar la red, suma para encontrar el broadcast.

1. Anatomía de una dirección IPv4

Una dirección IPv4 tiene 32 bits organizados en 4 grupos de 8 bits (octetos), separados por puntos en notación decimal.

Ejemplo: 192 . 168 . 10 . 130
192
168
10
130

La dirección se divide en dos partes según la máscara de subred:

Bits de red (primeros n bits) → identifican la red o subred
Bits de host (bits restantes) → identifican al dispositivo dentro de la red

Notación CIDR

El prefijo /n indica cuántos bits son de red. Ejemplo: 192.168.10.130/26 significa que los primeros 26 bits son de red y los últimos 6 son de host.

192.168.10.130 /26 — 26 bits de red (azul) · 6 bits de host (gris)

2. La máscara de subred

La máscara tiene 1s consecutivos en la parte de red y 0s en la parte de host. Se escribe igual que una IP.

Máscara /26 = 255.255.255.192
255
255
255
192
Los 26 primeros bits son 1 (parte de red) → los últimos 6 son 0 (parte de host).
El octeto crítico es el 4.º: 11000000 = 192 en decimal.

3. La operación AND (ANDing)

AND es una operación bit a bit que nos da la dirección de red al comparar la IP con la máscara.

Regla: el resultado es 1 solo cuando los dos bits son 1. En cualquier otro caso es 0.

Ejemplo completo: 192.168.10.130 AND 255.255.255.192

El único octeto con acción interesante es el 4.º (los tres primeros dan el mismo resultado porque la máscara es 255):

Resultado: La dirección de red es 192.168.10.128.
Los bits de host quedaron en 0 porque la máscara los anuló con AND.

4. La Wildcard mask (complemento de la máscara)

La wildcard es el inverso de la máscara: tiene 0s donde la máscara tiene 1s, y 1s donde la máscara tiene 0s.

Fórmula: Wildcard = 255.255.255.255 − Máscara de subred

Cálculo para /26 (máscara 255.255.255.192)

1.er octeto 2.do octeto 3.er octeto 4.to octeto 255 255 255 255 − 255 255 255 192
= 0 0 0 63

Wildcard /26 = 0.0.0.63

¿Qué significa 63? Es el "tamaño" de la subred menos 1.
63 en binario = 00111111 → exactamente los 6 bits de host en 1.
La subred tiene 26 = 64 direcciones totales (0 a 63).

5. Broadcast = Red + Wildcard

Una vez que tienes la dirección de red y la wildcard, el broadcast se calcula sumando octeto a octeto:

📌 Dirección de Red 192.168.10.128
+ Wildcard 0.0.0.63
= Broadcast 192.168.10.191

La suma se hace octeto por octeto: 128 + 63 = 191. Los primeros tres octetos no cambian porque la wildcard es 0 en esos octetos.

Primer host, último host y cantidad de hosts

+1 Primer host = Red + 1 192.168.10.129
−1 Último host = Broadcast − 1 192.168.10.190
2h Hosts útiles = 26 − 2 62 hosts
Resumen de la subred 192.168.10.128/26
Red: 192.168.10.128  |  Broadcast: 192.168.10.191
Primer host: 192.168.10.129  |  Último host: 192.168.10.190
Hosts útiles: 62

8. Resumen del método (los 5 pasos)

  1. Convertir el prefijo /n a máscara de subred en decimal
  2. Calcular la wildcard: 255.255.255.255 − Máscara
  3. Aplicar AND entre la IP y la máscara → dirección de red
  4. Sumar wildcard a la red → broadcast. Luego ±1 para primer y último host
  5. Calcular hosts útiles: 2h − 2 (h = bits de host = 32 − prefijo)
